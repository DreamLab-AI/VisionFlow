- ## About SHA-256
  id:: sha256-about

  - **SHA-256** (Secure Hash Algorithm 256-bit) is a cryptographic [[Hash Function]] from the [[SHA-2]] family, designed by the [[NSA]] and standardised by [[NIST]] in [[FIPS 180-4]]. It produces a fixed 256-bit (32-byte) hash value from arbitrary-length input data, serving as the cryptographic foundation for [[Bitcoin]]'s [[Proof-of-Work]] [[Consensus Mechanism]], [[Merkle Tree]] construction, and [[Transaction]] identification.

  - ### Bitcoin Mining & Proof-of-Work
    id:: sha256-bitcoin-mining
    - **Double SHA-256 Algorithm**: [[Bitcoin]] uses double SHA-256 hashing (SHA256(SHA256(block_header))) for [[Mining]] to mitigate [[Length Extension Attack]]s and provide additional security margin.
    - **Block Header Hashing**: [[Miners]] compute SHA-256 hashes of 80-byte [[Block Header]]s containing [[Version]], [[Previous Block Hash]], [[Merkle Root]], [[Timestamp]], [[Difficulty]] [[Target]], and [[Nonce]].
    - **Nonce Iteration**: Miners increment the 32-bit [[Nonce]] field (4.29 billion values) and modify the [[Coinbase Transaction]] [[ExtraNonce]] to find a hash below the [[Difficulty Target]].
    - **Leading Zero Requirement**: Valid [[Block Hash]]es must have a specified number of leading zero bits (e.g., ~73 leading zeros for difficulty 50 trillion in 2025).
    - **Hash Rate Metrics**: Global Bitcoin network hash rate exceeded 600 EH/s (exahashes per second) in 2025, representing ~600 quintillion (6×10^20) SHA-256 computations per second.
    - **Difficulty Adjustment**: [[Bitcoin Protocol]] adjusts [[Mining Difficulty]] every 2016 blocks (~2 weeks) to maintain ~10-minute block intervals based on network hash rate.
    - **ASIC Mining Dominance**: Purpose-built [[ASIC]] (Application-Specific Integrated Circuit) miners like [[Bitmain]] [[Antminer]] S21 (200 TH/s) dominate Bitcoin mining, rendering [[CPU]] and [[GPU]] mining economically infeasible.
    - **Energy Consumption**: Bitcoin SHA-256 mining consumes ~150-200 TWh annually (2025), comparable to medium-sized nations, driving innovation in [[Renewable Energy]] integration.

  - ### Bitcoin Address Generation
    id:: sha256-address-generation
    - **Hash Pipeline**: [[Bitcoin Address]]es use SHA-256 + [[RIPEMD-160]] pipeline: `RIPEMD160(SHA256(public_key))` produces 160-bit hash for address generation.
    - **Base58Check Encoding**: The resulting hash undergoes [[Base58Check]] encoding with version byte and checksum (double SHA-256) to create human-readable addresses (e.g., `1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa` - [[Satoshi Nakamoto]]'s [[Genesis Block]] address).
    - **P2PKH Addresses**: [[Pay-to-Public-Key-Hash]] (P2PKH) addresses begin with "1" and use SHA-256 in the hashing pipeline.
    - **P2SH Addresses**: [[Pay-to-Script-Hash]] (P2SH) addresses (prefix "3") also use SHA-256 + RIPEMD-160 for script hash generation.
    - **SegWit Addresses**: [[Segregated Witness]] [[Bech32]] addresses use SHA-256 in witness program hashing.

  - ### Merkle Tree Construction
    id:: sha256-merkle-trees
    - **Transaction Hashing**: Each [[Transaction]] is hashed with double SHA-256 to produce a 256-bit [[TXID]] (transaction identifier).
    - **Hierarchical Hashing**: TXIDs are paired and hashed together recursively (SHA256(SHA256(tx1 || tx2))) to build the [[Merkle Tree]].
    - **Merkle Root**: The final hash at the tree's root is included in the [[Block Header]], providing cryptographic commitment to all transactions in the block.
    - **Efficient Verification**: [[SPV]] ([[Simplified Payment Verification]]) clients can verify transaction inclusion using only the [[Merkle Proof]] (log₂(n) hashes) rather than downloading full blocks.
    - **Block Integrity**: Changing any transaction invalidates the Merkle root and consequently the entire [[Block Hash]], making tampering computationally infeasible.

  - ### Hash Function Properties
    id:: sha256-properties
    - **1. Deterministic**: Same input always produces identical output hash.
    - **2. Avalanche Effect**: Changing a single input bit flips ~50% of output bits (128 bits on average), ensuring sensitivity to input changes.
    - **3. Preimage Resistance**: Computationally infeasible to find input that produces specific hash output (requires ~2^256 operations).
    - **4. Second Preimage Resistance**: Computationally infeasible to find different input producing same hash as given input.
    - **5. Collision Resistance**: Computationally infeasible to find two different inputs producing identical hash (requires ~2^128 operations due to [[Birthday Paradox]]).
    - **6. Fixed Output Size**: Always produces exactly 256 bits (64 hexadecimal characters) regardless of input length.
    - **7. Non-Invertible**: One-way function - cannot derive input from hash output (no inverse function exists).
    - **8. Computational Efficiency**: Fast to compute in forward direction (MB/s software throughput, TH/s with ASICs).

  - ### Algorithm Technical Details
    id:: sha256-algorithm
    - **Message Preprocessing**: Input padded to 512-bit multiple with 1-bit, zero-bits, and 64-bit length field.
    - **512-bit Blocks**: Message processed in 512-bit (64-byte) chunks sequentially.
    - **Message Schedule**: Each 512-bit block expanded into 64 x 32-bit words (W[0] through W[63]) using bitwise operations.
    - **64 Rounds**: Compression function performs 64 rounds of operations on eight 32-bit working variables (a, b, c, d, e, f, g, h).
    - **Initialization Vectors**: Eight 32-bit constants derived from first 32 bits of fractional parts of square roots of first 8 primes (2, 3, 5, 7, 11, 13, 17, 19).
    - **Round Constants**: 64 constants (K[0] through K[63]) derived from first 32 bits of fractional parts of cube roots of first 64 primes.
    - **Bitwise Operations**: Uses [[Bitwise Rotation]], [[XOR]], [[AND]], [[Addition Modulo 2^32]] operations in compression function.
    - **Davies-Meyer Construction**: Final hash computed by adding working variables to previous hash value (modulo 2^32).

  - ### Bitcoin Script Operations
    id:: sha256-script-ops
    - **OP_SHA256**: [[Bitcoin Script]] operation that computes single SHA-256 hash of top stack item.
    - **OP_HASH256**: Computes double SHA-256 (equivalent to SHA256(SHA256(x))) for Bitcoin-standard hashing.
    - **Script Verification**: Used in [[Smart Contract]]-like conditions within Bitcoin transactions (e.g., [[Hash Time Locked Contract]]s).
    - **Hash Puzzles**: SHA-256 enables [[Hash Lock]]s where funds are spendable only by revealing preimage of specific hash.

  - ### Bitcoin Hash Rate & Network Security
    id:: sha256-hash-rate
    - **2025 Hash Rate**: Global Bitcoin network exceeds 600 EH/s (exahashes per second), representing unprecedented computational power.
    - **Security Margin**: At 600 EH/s, attacking Bitcoin via [[51% Attack]] requires controlling >300 EH/s (~$15+ billion in hardware at 2025 ASIC prices).
    - **Hash Rate Growth**: Bitcoin hash rate grew from ~10 MH/s (2009) → 1 EH/s (2016) → 600+ EH/s (2025), demonstrating exponential network security growth.
    - **Difficulty Epoch**: Every 2016 blocks, [[Bitcoin Protocol]] adjusts difficulty based on actual block interval vs. target 10 minutes.
    - **Historical Difficulty**: Bitcoin difficulty increased from 1.0 (2009) to >80 trillion (2025), reflecting hash rate expansion.

  - ### ASIC Mining Hardware
    id:: sha256-asic-mining
    - **ASIC Architecture**: [[Application-Specific Integrated Circuit]]s designed exclusively for SHA-256 computation, featuring thousands of parallel hashing cores.
    - **Leading Manufacturers**: [[Bitmain]] (Antminer series), [[MicroBT]] (Whatsminer), [[Canaan]] (AvalonMiner), [[Ebang]].
    - **2025 Flagship Models**:
      - [[Antminer S21]]: ~200 TH/s, ~3500W, ~17.5 J/TH efficiency
      - [[Whatsminer M60S]]: ~180 TH/s, ~3400W, ~18.9 J/TH efficiency
      - [[AvalonMiner 1466]]: ~150 TH/s, ~3400W, ~22.7 J/TH efficiency
    - **Technology Nodes**: Modern ASICs use 5nm and 3nm [[Semiconductor]] processes for maximum energy efficiency.
    - **Hashboard Design**: Multiple hashing chips arranged on hashboards with dedicated cooling (air or [[Immersion Cooling]]).
    - **Economic Viability**: ASIC profitability depends on [[Electricity Cost]], [[Bitcoin Price]], [[Network Difficulty]], and hardware efficiency.
    - **Obsolescence Cycle**: ASIC hardware typically becomes economically obsolete in 2-4 years due to efficiency improvements and difficulty increases.

  - ### Other Blockchain Implementations
    id:: sha256-other-blockchains
    - **Bitcoin Cash (BCH)**: Uses identical SHA-256 Proof-of-Work, enabling [[Hash Rate]] sharing/switching with Bitcoin based on profitability.
    - **Bitcoin SV (BSV)**: Maintains SHA-256 mining algorithm from Bitcoin Cash fork.
    - **Namecoin**: First Bitcoin fork (2011) using SHA-256 for merged mining with Bitcoin.
    - **Litecoin**: Uses [[Scrypt]] algorithm which internally calls SHA-256 as part of its memory-hard hashing.
    - **Merged Mining**: [[Namecoin]], [[Elastos]], and other chains enable simultaneous mining with Bitcoin using same SHA-256 work.
    - **Mining Pool Switching**: SHA-256 miners can switch between Bitcoin, Bitcoin Cash, and BSV based on [[Profitability]] algorithms.

  - ### SHA-2 Family Comparison
    id:: sha256-family
    - **SHA-224**: Truncated version producing 224-bit output (uses same algorithm, discards 32 bits).
    - **SHA-256**: Standard 256-bit version, most widely adopted in [[Blockchain]] and [[Cryptography]].
    - **SHA-384**: Uses 64-bit words, 384-bit output (truncated SHA-512).
    - **SHA-512**: Uses 64-bit words, 512-bit output, more efficient on 64-bit processors.
    - **SHA-512/224, SHA-512/256**: SHA-512 variants with truncated output, offering better performance on 64-bit systems than SHA-224/256.
    - **Common Design**: All SHA-2 variants share [[Merkle-Damgård]] structure with variant-specific parameters.

  - ### SHA-256 vs SHA-3 (Keccak)
    id:: sha256-vs-sha3
    - **Design Philosophy**: SHA-256 uses Merkle-Damgård construction; [[SHA-3]] ([[Keccak-256]]) uses [[Sponge Construction]].
    - **Ethereum Choice**: [[Ethereum]] selected Keccak-256 (variant of SHA-3) for hashing to differentiate from Bitcoin and provide ASIC resistance initially.
    - **Security**: Both provide ~128-bit collision resistance; Keccak offers additional flexibility and NIST considered it more future-proof.
    - **Performance**: SHA-256 generally faster in hardware acceleration; Keccak more efficient in software on some platforms.
    - **Length Extension**: SHA-256 vulnerable to [[Length Extension Attack]]s (mitigated by double hashing); Keccak immune due to sponge design.
    - **Standardization**: SHA-256 standardised 2001 (FIPS 180-2); SHA-3 standardised 2015 (FIPS 202).

  - ### Security Analysis & Cryptanalysis
    id:: sha256-security
    - **No Practical Collisions**: Zero collision attacks demonstrated on full SHA-256 after 20+ years of cryptanalysis (as of 2025).
    - **No Preimage Attacks**: No practical preimage or second preimage attacks on full 64-round SHA-256.
    - **Reduced-Round Attacks**: Academic attacks exist on reduced rounds (e.g., 52/64 rounds) but are impractical and don't threaten real-world usage.
    - **Quantum Computing Threat**: [[Grover's Algorithm]] reduces preimage security from 2^256 to 2^128 operations (still secure but reduced margin).
    - **Collision Resistance**: [[Birthday Paradox]] theoretically enables collisions at ~2^128 operations, but no practical method exists.
    - **Birthday Attack**: Finding collision requires ~2^128 hash computations (infeasible with current/foreseeable technology).
    - **Length Extension Vulnerability**: SHA-256 susceptible to length extension attacks; Bitcoin mitigates by using double SHA-256.
    - **Side-Channel Resistance**: Implementations must protect against [[Timing Attack]]s, [[Power Analysis]], and [[Cache-Timing Attack]]s.

  - ### Performance & Optimization
    id:: sha256-performance
    - **CPU Software**: Modern CPUs achieve ~100-500 MB/s SHA-256 throughput with optimised implementations.
    - **Hardware Acceleration**: [[Intel SHA Extensions]] (SHA-NI) provide ~2-3× speedup on supported CPUs (Intel 2013+, AMD Ryzen).
    - **GPU Mining**: [[NVIDIA]] RTX 4090 achieves ~1-3 GH/s SHA-256 (obsolete for Bitcoin but useful for other applications).
    - **FPGA Implementation**: Field-Programmable Gate Arrays achieve ~5-20 GH/s with custom designs.
    - **ASIC Performance**: Modern Bitcoin ASICs deliver 100-200+ TH/s (100,000-200,000 GH/s) per unit.
    - **Energy Efficiency**: ASIC efficiency improved from ~1000 J/TH (2013) to ~15-20 J/TH (2025), representing 50× improvement.
    - **Parallel Processing**: SHA-256 highly parallelizable, enabling massive-scale concurrent computation in mining.
    - **Memory Requirements**: Minimal memory footprint (~256 bytes working state), unlike memory-hard algorithms like [[Scrypt]] or [[Argon2]].

  - ### Data Integrity & Authentication
    id:: sha256-data-integrity
    - **File Verification**: SHA-256 checksums verify software downloads, OS images, and file integrity (e.g., Linux ISO verification).
    - **Digital Signatures**: [[ECDSA]], [[RSA]], and other signature schemes hash messages with SHA-256 before signing.
    - **TLS/SSL Certificates**: [[X.509]] certificates use SHA-256 for signature generation (migrated from deprecated [[SHA-1]]).
    - **HMAC Construction**: [[HMAC-SHA-256]] provides message authentication codes for secure communication protocols.
    - **Password Hashing**: SHA-256 sometimes used for password hashing (though [[Argon2]], [[bcrypt]], [[scrypt]] preferred due to memory-hardness and key stretching).
    - **Git Version Control**: [[Git]] uses SHA-1 (migrating to SHA-256) for commit and object identification.
    - **Docker Images**: Container image layers identified by SHA-256 hashes for integrity verification.

  - ### Quantum Computing Resistance
    id:: sha256-quantum
    - **Grover's Algorithm**: Quantum computers using [[Grover's Algorithm]] could find preimages in ~2^128 operations instead of 2^256.
    - **Collision Search**: Quantum [[BHT Algorithm]] (Brassard-Høyer-Tapp) could find collisions in ~2^85 operations versus 2^128 classical.
    - **Current Status**: No quantum computer (2025) approaches scale needed to threaten SHA-256 (requires millions of logical qubits).
    - **Security Margin**: Even with quantum advantage, 2^128 operations remain impractical for foreseeable future.
    - **Post-Quantum Planning**: NIST recommends SHA-256 acceptable for near-term use but suggests SHA-384 or SHA-512 for long-term post-quantum applications.
    - **Bitcoin Implications**: Quantum threat to SHA-256 mining significantly less severe than threat to [[ECDSA]] signature scheme used for Bitcoin private keys.

  - ### Implementation Considerations

## Academic Context

- SHA-256 is a cryptographic hash function belonging to the [[SHA-2]] family, designed by the [[National Security Agency]] (NSA) to produce a fixed 256-bit output from arbitrary input data.
  - It was developed to address vulnerabilities found in earlier hash functions such as [[MD5]] and [[SHA-1]], which suffered from collision attacks and preimage weaknesses.
  - The algorithm's design incorporates [[Merkle-Damgård Construction]] with strengthening to resist [[Length Extension Attack]]s, enhancing its security profile.
- Over two decades of cryptanalysis have confirmed SHA-256's robustness, with no practical collisions or preimage attacks demonstrated to date (as of 2025).
  - Its security foundations rest on [[Collision Resistance]], [[Preimage Resistance]], and [[Second Preimage Resistance]], making it suitable for integrity verification and authentication.
  - Academic research consistently highlights SHA-256's balance between cryptographic strength and computational efficiency, making it a standard in modern cryptography and blockchain systems.

## Current Landscape (2025)

- SHA-256 remains the cryptographic foundation of [[Bitcoin]] and numerous blockchain networks, securing over $1.5 trillion in digital asset value.
  - The Bitcoin network's hash rate exceeds 600 EH/s, representing the largest computational network in human history dedicated to SHA-256 computation.
  - [[ASIC]] mining hardware has evolved from early [[CPU]]/[[GPU]] mining to specialized 5nm and 3nm chips delivering 200+ TH/s at 15-20 J/TH efficiency.
- SHA-256 underpins internet security infrastructure beyond blockchain:
